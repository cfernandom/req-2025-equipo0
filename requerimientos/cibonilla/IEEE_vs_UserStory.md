# Comparacion de redacción de requerimientos(IEEE vs UserStory)

| IEEE                                                                                                            | UserStory(Ágil)                                                                                                                                              |
|-----------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------|
| El sistema deberá permitir el registro de ingresos y gastos de manera manual mediante formularios               | Como estudiante que derrocha mucho dinero quiero tener un registro de mis ingresos y gastos para ahorrar lo suficiente para alcanzar mis metas.              |
| El sistema deberá generar notificaciones de alerta cuando el usuario pase los limites de gasto                  | Como estudiante que gasta dinero sin pensar mucho quiero tener un recordatorio de los limites de  gasto que tengo para evitar consumos excesivos             |
| El sistema deberá permitir que el usuario configure las metas de ahorro                                         | Como estudiante que quiere cumplir sus sueños quiero poder colocarme mis propias metas de ahorro  para comprarme las cosas que quiero y necesito.            |
| El sistema deberá generar sugerencias de ahorro dependiendo de los hábitos de consumo del usuario               | Como estudiante que no tiene mucha educación financiera quiero tener sugerencias que me ayuden a mejorar mis hábitos de consumo y por ende mejorar mi ahorro |
| El sistema deberá generar notificaciones de recordatorio de pagos cuando las fechas esten proximas a una semana | Como estudiante que es olvidadizo quiero tener un sistema que me recuerde los pagos que tego que realizar con un tiempo prudente                             |
